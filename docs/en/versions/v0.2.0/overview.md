# v0.2.0 Overview

Version 0.2.0 is the first dense draft after the initial public release.

It adds three missing pillars:

1. **Discovery** - how descriptions should help packs trigger or select correctly.
2. **Maintenance** - how scripts, skills, and tools should update knowledge without collapsing it into instructions.
3. **Evaluation** - how to measure discovery, grounding, and answer quality over time.

It also expands the Karpathy LLM Wiki reference, client integration guidance, and versioned snapshots.

## Highlights

- Stronger Skills vs Knowledge boundary.
- Explicit description optimization guidance.
- Knowledge-pack discovery evals.
- Maintenance automation guidance.
- Client trust, activation, and fenced-context rules.
- Complete LLM Wiki pattern mapping.
- Copy Markdown support on the documentation site.
- Multi-language docs and version snapshots.
